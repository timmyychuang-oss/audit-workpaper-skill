# 审计底稿写法参考范例

本文件为虚构场景，展示12列和7列的**实际写法风格和详细程度**。
生成新底稿时参照此文件的写法，不要照搬内容。

---

## 一、总览范例（Sheet 1）

```
审计对象：某智能硬件公司的配件供应链
业务描述：公司生产智能门锁，核心配件（指纹模组、主控芯片、锌合金外壳）
         由3-4家供应商提供，公司负责组装和全国经销商配送。
业务阶段：快速扩张（月出货从5万台增长至15万台）
审计范围：从配件采购到经销商签收的供应链全流程

战略目标：
1. 产能匹配：配件供应跟上出货增长节奏
2. 成本控制：配件采购和物流成本在预算范围内
3. 质量保障：来料合格率和成品良率达标
4. 供应安全：不因单一供应商问题导致停产

底线类型：战略底线 / 财务底线
```

## 二、底线风险12列范例（Sheet 2）

### BLR-1 产能瓶颈（需求预测与供应商产能失配）

| 列 | 内容 |
|----|------|
| 1 编号 | BLR-1 |
| 2 名称 | 产能瓶颈（需求与供应失配） |
| 3 底线类型 | 战略底线 |
| 4 关联目标 | 产能匹配 + 成本控制 |
| 5 风险描述 | 出货量从5万台/月增至15万台/月，配件需求三倍增长。如果需求预测偏高，库存积压占用资金；如果偏低，紧急采购成本上升20-30%。真正痛点不是偶尔预测不准，而是偏差没有被及时发现，导致每月重复犯同样的错误。 |
| 6 逻辑链 | 需求预测偏差持续存在 → 紧急采购增加（成本上升20-30%）或库存积压（资金占用）→ 出货计划延迟或资金效率下降 → 经销商交付不及时 → 产能匹配目标失败 |
| 7 底线标准 | 需求预测MAPE ≤ 30%（超出=预测体系需改进）[L2: APICS基准] &#124; 紧急订单占比 ≤ 15%（超出=计划系统性失效）[L5: 专业判断] &#124; 呆滞库存(>90天)占比 ≤ 5% [L5] &#124; 库存周转天数 ≤ 30天 [L5] |
| 8 失控后果 | 假设月采购额￥600万：紧急订单溢价20% × 超标部分15% × 12月 = ￥216万/年；呆滞库存5%超标 → 资金沉淀￥360万 |
| 9 行业先例 | 某消费电子代工厂因指纹模组预测偏差，3个月库存积压，报废损失约￥500万 [来源：行业访谈，待确认] |
| 10 行业水位+合规 | 行业水位：成熟消费电子供应链MAPE通常15-25%。合规定位：最佳实践（非强制要求） |
| 11 替代方案 | 最基础（零成本）：建立月度需求vs实际复盘会议，追踪偏差趋势 &#124; 进阶：统计预测模型+偏差自动预警 &#124; 标杆：基于经销商POS数据的需求感知系统 |
| 12 补偿控制 | ① 采购团队经验补救紧急需求（→ 验证方向：分析紧急订单中"经验补救"比例，见T1.2） ② 供应商有弹性产能（→ 验证方向：检查供应商产能利用率，见T2.2） |

### BLR-3 结算资金损失（对账机制缺陷）

| 列 | 内容 |
|----|------|
| 1 编号 | BLR-3 |
| 2 名称 | 结算资金损失（对账机制缺陷） |
| 3 底线类型 | 财务底线 |
| 4 关联目标 | 成本控制 |
| 5 风险描述 | 配件按件结算，如果到货验收只签字不清点，结算以供应商发货单为准而非我方验收，单笔差异小但月度放大后可能达数十万级。辅料（螺丝/包装/标签）更容易藏差异——单价低、关注度低、量大。 |
| 6 逻辑链 | 到货验收流于形式 → 结算数量以供应商数据为准 → 单笔差异小不触发关注 → 月度百万级采购额放大差异 → 累积资金损失 → 成本控制目标落空 |
| 7 底线标准 | 结算数量与验收数量一致率 = 100% [L1: 财务基本要求] &#124; 结算单价与合同单价一致率 = 100% [L1] &#124; 全链路数量守恒差异 < 0.5% [L5] &#124; 三单匹配率 ≥ 99% [L5] |
| 8 失控后果 | 假设月结算额￥600万：0.5%守恒差异 × 12月 = ￥36万/年；辅料溢价如超15% → 额外数十万 |
| 9 行业先例 | 制造业常见：某代工厂因辅料用量未独立验证，年度多付超￥150万 [来源：审计行业案例，待确认] |
| 10 行业水位+合规 | 行业水位：成熟企业三单匹配率>99.5%。合规定位：基本要求（财务审计必查项） |
| 11 替代方案 | 最基础（零成本）：验收单增加清点数量列 &#124; 进阶：ERP自动三单匹配+差异预警 &#124; 标杆：系统内置全链路数量守恒校验 |
| 12 补偿控制 | ① 供应商长期合作降低故意差错概率（→ 见T3.1） ② 财务月度对账可能捕获大额差异（→ 见T3.3） |

## 三、测试方案7列范例（Sheet 3）

### T1.1 需求预测准确率分析

| 列 | 内容 |
|----|------|
| A 审计目标 | 配件需求预测到底准不准？业务团队的月度需求计划和实际采购量之间差距有多大？ |
| B 证据源 | 主要：需求计划表 + 采购申请明细表。辅助：历史预测偏差记录 |
| C 分析步骤 | 第一步：从需求计划表提取每月每种配件的预测数量。第二步：从采购申请表提取同月同配件的实际申请数量。第三步：计算偏差 = (预测-实际)/实际 × 100%。第四步：计算MAPE，按月趋势画图。第五步：计算Bias方向——持续为正=系统性高估。第六步：按配件类型分组——是全部都偏差大，还是某种配件特别不准？第七步：识别异常月份——偏差突然飙高时对应什么业务事件。 |
| D 预期输出 | 月度MAPE趋势图（折线图）&#124; Bias趋势图 &#124; 分配件预测准确率排名 &#124; 异常月份清单 |
| E 判断标准 | MAPE > 30% = 预测体系需改进 &#124; MAPE > 50% = 预测几乎没有参考价值 &#124; Bias持续同方向 > 3个月 = 存在系统性偏差 &#124; 某配件MAPE显著高于均值 = 需专项关注 |
| F 证据支撑度 | ★★★★★ 数据充分 |
| G 关联BLR | BLR-1 |

### T3.1 结算数量一致性验证

| 列 | 内容 |
|----|------|
| A 审计目标 | 结算数量是否以我方验收数量为准？结算的事实基础应该是"我方确认收到多少"，而非"供应商声称发了多少"。 |
| B 证据源 | 主要：结算清单 + 入库验收记录。辅助：合同单价表 |
| C 分析步骤 | 第一步：从结算清单提取每笔结算数量和金额。第二步：从入库记录提取对应批次的验收数量。第三步：逐笔比对——结算数量 vs 验收数量。差异A：结算>验收=多付风险。第四步：计算总体差异率。第五步：按供应商分组——哪个差异最大？第六步：追溯差异原因——来自发货环节还是结算环节？第七步：评估验收机制——是逐件清点还是按送货单签收？ |
| D 预期输出 | 逐笔一致性比对清单 &#124; 差异率统计 &#124; 按供应商差异分布 &#124; 多付风险金额汇总 |
| E 判断标准 | 结算≠验收 = 对账缺陷（不论差异率大小）&#124; 差异率 > 0.5% = 系统性问题 &#124; 多付风险 > ￥5万/月 = 需升级汇报 &#124; 验收方式为"按送货单签收" = 机制本身需改进 |
| F 证据支撑度 | ★★★★★ 数据充分 |
| G 关联BLR | BLR-3 |
