## 跨阶段一致性检查（S6输出前执行）

在生成最终Excel前，执行以下8项全局检查。**任何一项不通过，必须修复后再输出。**

### 检查1: 目标→BLR覆盖
```
遍历S1的每个战略目标：
  该目标是否被至少1个BLR的列4引用？
  如果否 → 追问用户"目标X没有关联的底线风险，是否遗漏？"
```

### 检查2: BLR→测试覆盖
```
遍历S2的每个BLR：
  该BLR是否被至少1个测试的列G引用？
  如果否 → 提示"BLR-N没有对应测试，需要补充设计"
```

### 检查3: 测试→证据源覆盖
```
遍历S3的每个测试：
  该测试是否在Sheet 4中有至少1个证据源？
  如果否 → 提示"测试T-N.M没有证据源，需要补充映射"
```

### 检查4: 证据源→测试反向映射
```
遍历Sheet 4的每个证据源：
  该证据源的"审计用途"列是否有测试编号？
  如果否 → 删除该证据源（无用证据不应出现在底稿中）
```

### 检查5: 外部方数据独立性
```
遍历Sheet 5中来源为"被审方提供"或"供应商提供"的证据源：
  是否有"独立验证方法"？
  如果否 → 标注为"⚠ 对账盲区"，在Sheet 6中增加待办项
```

### 检查6: P级一致性
```
遍历Sheet 6的每个待办项：
  待办P级 ≥ 其关联BLR中最高的P级？
  如果否 → 调整待办P级
```

### 检查7: 列12回填
```
遍历S2的每个BLR列12：
  "验证方向"是否已更新为测试编号（如"→ 见T1.2"）？
  如果否 → 执行回填
```

### 检查8: 全链路追溯
```
随机抽取1个Sheet 6待办项：
  待办 → 对应测试编号 → 对应BLR编号 → 对应战略目标
  链路完整？每步引用有效？
  如果断裂 → 修复断链
```

---

## 一致性修复流程

1. 向用户报告不一致项
2. 提出修复建议
3. 用户确认修复方案
4. 执行修复
5. 重新运行受影响的检查项
6. 全部通过后才输出Excel
