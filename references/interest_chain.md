# 利益链条思考 / Interest Chain Thinking

这是一个**补充视角**，不是必填模板。在S2推导完BLR后，用这组追问审视每个风险的深层结构。

---

## 三个追问

对每个BLR，依次问：

### 1. 收益和成本在谁身上？

> "这个风险里，谁获利了？谁承担了成本？是不是同一个人？"

如果获利方和受损方不是同一个人——这本身就是一个结构性信号。获利方没有动力去管控风险，因为成本不由他承担。

### 2. 有没有反馈机制？

> "受损的信号能传回获利方吗？有什么机制让获利方感受到代价？"

如果没有反馈机制（没有投诉渠道、没有考核挂钩、没有监管威慑），风险就会**自我放大**——获利方持续获利，受损方持续受损，直到某个外部事件引爆。

**收益-受损不对称 + 无反馈机制 = 底线风险的结构性来源。**

### 3. 谁是关键节点？

> "有没有某个人或角色，同时连接了多条利益链？"

关键节点可能是放大器（他的一个决策同时影响多方），也可能是制动器（他是唯一能阻止风险的人）。找到关键节点 = 找到审计的着力点。

## 思考示范：互联网医疗平台"先药后方"

**不是标注"S4→S1"，而是沿着"为什么"追问下去：**

**平台为什么要先发药再补处方？**
→ 因为发货速度是用户体验核心指标，直接影响复购率
→ 复购率影响GMV，GMV是投资人看的核心数字
→ 投资人的回报预期在间接驱动这个行为

获利方 = 平台（GMV↑）+ 投资人（估值↑）
受损方 = 患者（未经审核的处方）+ 药师（执业风险）

**药师为什么不拦？**
→ 药师是平台雇的，日均审方量远超上限
→ 拦一单 = 平台少一单收入
→ 药师承担执业风险，但平台承担业绩压力——风险和收益不在同一个人身上

**有反馈机制吗？**
→ 患者不知道处方是后补的（无感知 = 无投诉 = 无反馈）
→ 药监追溯码2026年才全量采集（监管反馈滞后）
→ 药师即使发现问题也没有权力叫停发货流程

没有反馈 → 风险持续放大 → 直到追溯码上线成为引爆点

**关键节点是谁？**
→ 平台技术架构的设计者——他决定了"处方审核完成"是不是发货的前置条件
→ 如果架构上没有硬卡点，所有下游环节（药师、物流）都无法拦截

## 怎么用

追问的发现不需要写进新的列。它可以：
- 加深**列5（风险描述）**中"真正的痛点"段落
- 让**列6（逻辑链）**不只是事件因果，还包含利益驱动
- 让**列12（补偿性控制）**追问更深："没出事是因为有反馈机制在拦截，还是纯粹没爆发？"
- 影响**P级判断**：收益-受损不对称且无反馈机制的BLR，应考虑上调P级
